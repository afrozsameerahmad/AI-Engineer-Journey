# Day 29: LeetCode Question Practice

## Overview
This day emphasizes search patterns and problem-solving templates used frequently in LeetCode practice. The focus is on binary search (templates and variants), two-pointer techniques, sliding-window strategies, and hashmap/frequency-based solutions.

## Notebook / Practice Files
- `practice/1st.ipynb` — early practice problems
- `practice/2nd.ipynb` — intermediate problems
- `practice/3rd.ipynb` — additional practice
- `practice/4th.ipynb` — final practice set
- `binary_search/binary_search.ipynb` — binary search iterative & recursive templates

## What Was Practiced
- Implementing binary search (iterative and recursive) and its variants (`lower_bound`, `upper_bound`).
- Applying two-pointer patterns for pair-sum and partitioning problems.
- Sliding-window techniques for subarray/substring problems.
- Hashmap/frequency patterns for complement and counting problems.

## Complexity Snapshot

| Problem | Approach | Time | Space |
|---|---|---:|---:|
| Basic search | Binary search (iterative) | O(log n) | O(1) |
| Predicate search | Binary search (decision) | O(log n) | O(1) |

## Key Patterns Practiced
1. Binary search templates (index-finding and predicate-based).
2. Two-pointer and sorting reductions for pair/sum problems.
3. Sliding-window for variable-length subarray problems.
4. Hashmap for O(1) expected lookups and frequency tracking.

Medium — core algorithmic patterns needed for interview-style problems.
