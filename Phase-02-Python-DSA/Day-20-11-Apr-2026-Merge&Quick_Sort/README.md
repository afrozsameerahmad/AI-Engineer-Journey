# Day 20: Merge Sort and Quick Sort

## Overview
This day focuses on divide-and-conquer sorting algorithms and one warm-up array problem:

- Merge Sort
- Quick Sort (simple recursive + in-place partition version)
- Largest Element in an Array

## Exercises
The `exercises/` folder contains:

- `merge_sort.ipynb`: Merge helper + full recursive merge sort
- `quick_sort.ipynb`: Two quick sort styles (new-list recursive and in-place partition)
- `largest_ele.ipynb`: Linear scan to find the maximum value in an array

## Learning Objectives
- Understand divide-and-conquer strategy in sorting
- Implement merge sort and analyze why it is stable and consistently efficient
- Implement quick sort and understand pivot/partition behavior
- Compare merge sort vs quick sort trade-offs
- Practice time and space complexity analysis

## Complexity Snapshot
- Merge Sort: Time $O(n \log n)$, Space $O(n)$
- Quick Sort: Average Time $O(n \log n)$, Worst Time $O(n^2)$
- Largest Element Scan: Time $O(n)$, Space $O(1)$
