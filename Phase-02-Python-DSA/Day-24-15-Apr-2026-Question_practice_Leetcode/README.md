# Day 24: LeetCode Question Practice

## Overview
This day focuses on three classic array problems practiced in notebooks:

- Max Consecutive Ones
- Missing Number
- Two Sum

These exercises continue the pattern of writing a brute-force approach first and then improving it to a more optimal solution.

## Notebook Files
- `max_consecutive_ones.ipynb`: Count longest streak of consecutive 1s in a binary array.
- `missing_num.ipynb`: Three approaches - brute force, dictionary marking, and math sum formula.
- `two_sum_leetcode.ipynb`: Brute force nested loops and optimized hash-map solution.

## What Was Practiced
- Streak counting in arrays
- Presence checking via dictionary/hash map
- Pair-sum lookup using complement logic
- Comparing brute-force vs optimized complexity

## Complexity Snapshot (As Written in Notebooks)

| Problem | Approach | Time | Space |
|---|---|---|---|
| Max Consecutive Ones | Single pass count | O(n) | O(1) |
| Missing Number | Brute force (`in` check in loop) | O(n^2) | O(1) |
| Missing Number | Dictionary marking | O(n) | O(n) |
| Missing Number | Math sum formula | O(n) | O(1) |
| Two Sum | Brute force nested loops | O(n^2) | O(1) |
| Two Sum | Hash map (complement lookup) | O(n) | O(n) |

## Key Patterns Practiced
1. Linear traversal
2. Frequency/presence mapping with hash structures
3. Complement-based lookup
4. Brute-force to optimal refinement

## Difficulty Level
Easy to Easy-Medium interview-level array problems.
