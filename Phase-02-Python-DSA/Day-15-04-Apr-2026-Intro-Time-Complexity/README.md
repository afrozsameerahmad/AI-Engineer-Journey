## Key Learning Outcomes
After completing Day 15, I should be able to:
1. Explain what time complexity means in simple terms.
2. Use Big-O notation for basic algorithms.
3. Analyze complexity of single and nested loops.
4. Distinguish between `O(1)`, `O(log n)`, `O(n)`, and `O(n^2)`.
5. Understand worst-case focused analysis for DSA practice.
6. Recognize basic use of `Ω` and `Θ` notations.

## Quick Revision
- Time complexity measures growth of operations with input size.
- Big-O communicates asymptotic upper-bound behavior.
- Ignore constants and lower-order terms in Big-O.
- Loop structure is a fast way to estimate complexity.
- Better growth rate usually means better scalability.

## Suggested Practice
1. Write 5 loop snippets and compute their Big-O.
2. Compare linear search and binary search complexity.
3. Convert one nested-loop solution into a better approach.
4. Track operation counts manually for small and large `n`.
